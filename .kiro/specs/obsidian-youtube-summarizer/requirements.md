# 요구사항 문서

## 소개

옵시디언(Obsidian) 플러그인으로, 유튜브 영상 링크를 입력하면 해당 영상을 임베딩하고 AI(AWS Bedrock)를 활용하여 영상 내용을 자동으로 요약 정리하여 노트로 저장하는 기능을 제공한다. 사용자는 사이드바에서 유튜브 링크를 입력하고 "요약" 버튼을 눌러 간편하게 영상 요약 노트를 생성할 수 있다. 플러그인은 TypeScript로 개발하며, 자막 데이터가 없는 영상의 경우에도 영상 메타데이터(제목, 설명 등)를 활용한 대체 요약을 지원한다.

## 용어 정의 (Glossary)

- **플러그인(Plugin)**: 옵시디언에 추가 기능을 제공하는 확장 모듈
- **사이드바_패널(Sidebar_Panel)**: 옵시디언 사이드바 최상단에 표시되는 플러그인 전용 UI 영역
- **유튜브_링크_입력창(YouTube_URL_Input)**: 사이드바 패널 내에서 유튜브 영상 URL을 입력받는 텍스트 필드
- **요약_버튼(Summarize_Button)**: 유튜브 영상 요약 프로세스를 시작하는 버튼
- **요약_노트(Summary_Note)**: 유튜브 영상 임베딩과 AI 요약 내용이 포함된 옵시디언 마크다운 문서
- **영상_제목(Video_Title)**: 유튜브 API 또는 페이지 메타데이터에서 가져온 영상의 원본 제목
- **요약_프롬프트(Summary_Prompt)**: AI에게 영상 내용 요약을 요청할 때 사용하는 사전 저장된 프롬프트 템플릿
- **영상_자막(Video_Transcript)**: 유튜브 영상의 자막 또는 캡션 텍스트 데이터
- **플러그인_설정(Plugin_Settings)**: 플러그인의 동작을 사용자가 커스터마이즈할 수 있는 설정 화면
- **영상_설명(Video_Description)**: 유튜브 영상 페이지에서 가져온 영상의 설명 텍스트
- **AWS_Bedrock**: Amazon Web Services에서 제공하는 관리형 AI 모델 서비스로, 플러그인의 요약 기능에 사용되는 AI 백엔드
- **Bedrock_모델_ID(Bedrock_Model_ID)**: AWS Bedrock에서 사용할 AI 모델을 식별하는 고유 ID (예: anthropic.claude-3-sonnet)
- **AWS_리전(AWS_Region)**: AWS Bedrock API를 호출할 AWS 리전 (예: us-east-1)
- **대체_요약(Fallback_Summary)**: 영상_자막을 가져올 수 없는 경우, 영상_제목과 영상_설명 등 메타데이터를 기반으로 생성하는 요약

## 요구사항

### 요구사항 1: 사이드바 패널 표시

**사용자 스토리:** 개발자/사용자로서, 옵시디언 사이드바에서 유튜브 요약 기능에 빠르게 접근하고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 플러그인이 활성화되면, THE 플러그인 SHALL 옵시디언 좌측 사이드바에 전용 아이콘을 등록한다
2. WHEN 사용자가 사이드바 아이콘을 클릭하면, THE 사이드바_패널 SHALL 유튜브_링크_입력창과 요약_버튼을 포함한 패널을 표시한다
3. THE 유튜브_링크_입력창 SHALL 플레이스홀더 텍스트로 "유튜브 링크를 입력하세요"를 표시한다

### 요구사항 2: 유튜브 링크 유효성 검증

**사용자 스토리:** 사용자로서, 잘못된 링크를 입력했을 때 명확한 안내를 받고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 사용자가 요약_버튼을 클릭하면, THE 플러그인 SHALL 유튜브_링크_입력창의 값이 유효한 유튜브 URL 형식인지 검증한다
2. IF 입력된 URL이 유효한 유튜브 URL 형식이 아니면, THEN THE 플러그인 SHALL "유효한 유튜브 링크를 입력해주세요"라는 오류 메시지를 표시한다
3. THE 플러그인 SHALL youtube.com/watch, youtu.be, youtube.com/shorts 형식의 URL을 유효한 유튜브 URL로 인식한다
4. IF 유튜브_링크_입력창이 비어있는 상태에서 요약_버튼을 클릭하면, THEN THE 플러그인 SHALL "유튜브 링크를 입력해주세요"라는 오류 메시지를 표시한다

### 요구사항 3: 유튜브 영상 메타데이터 가져오기

**사용자 스토리:** 사용자로서, 영상 제목이 자동으로 노트 제목에 반영되길 원한다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 유효한 유튜브 URL이 제출되면, THE 플러그인 SHALL 해당 영상의 영상_제목을 가져온다
2. WHEN 유효한 유튜브 URL이 제출되면, THE 플러그인 SHALL 해당 영상의 영상_자막 데이터를 가져온다
3. WHEN 유효한 유튜브 URL이 제출되면, THE 플러그인 SHALL 해당 영상의 영상_설명 데이터를 가져온다
4. IF 영상_제목을 가져오는 데 실패하면, THEN THE 플러그인 SHALL 영상 ID를 기반으로 대체 제목을 생성한다
5. IF 영상_자막을 가져오는 데 실패하면, THEN THE 플러그인 SHALL 영상_제목과 영상_설명을 대체 데이터로 사용하여 요약 프로세스를 계속 진행한다
6. IF 영상_자막을 가져오는 데 실패하면, THEN THE 사이드바_패널 SHALL "자막을 가져올 수 없어 영상 설명 기반으로 요약합니다"라는 안내 메시지를 표시한다

### 요구사항 4: 요약 노트 생성

**사용자 스토리:** 사용자로서, 유튜브 영상이 임베딩되고 요약이 정리된 노트를 자동으로 생성하고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 영상_자막 데이터 또는 대체 메타데이터가 준비되면, THE 플러그인 SHALL 새로운 마크다운 문서를 생성한다
2. THE 요약_노트 SHALL 문서 제목으로 영상_제목을 사용한다
3. THE 요약_노트 SHALL 문서 상단에 옵시디언 네이티브 임베딩 문법(`![](URL)`)을 사용하여 유튜브 영상을 임베딩한다
4. WHEN 요약_노트가 생성되면, THE 플러그인 SHALL 해당 노트를 옵시디언 편집기에서 자동으로 열어 표시한다
5. IF 동일한 영상_제목의 노트가 이미 존재하면, THEN THE 플러그인 SHALL 파일명에 타임스탬프를 추가하여 중복을 방지한다

### 요구사항 5: AI 기반 영상 내용 요약

**사용자 스토리:** 사용자로서, 영상의 핵심 내용이 자동으로 정리되어 노트에 포함되길 원한다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 영상_자막 데이터가 준비되면, THE 플러그인 SHALL 저장된 요약_프롬프트와 영상_자막을 결합하여 AWS_Bedrock에 요약을 요청한다
2. WHEN 영상_자막이 없고 대체 메타데이터가 준비되면, THE 플러그인 SHALL 저장된 요약_프롬프트와 영상_제목 및 영상_설명을 결합하여 AWS_Bedrock에 대체_요약을 요청한다
3. THE 요약_노트 SHALL 영상 임베딩 아래에 AWS_Bedrock이 생성한 요약 내용을 마크다운 형식으로 포함한다
4. WHEN 대체_요약이 사용된 경우, THE 요약_노트 SHALL 요약 상단에 "이 요약은 자막이 아닌 영상 설명을 기반으로 생성되었습니다"라는 안내 문구를 포함한다
5. WHILE AI 요약이 진행 중인 동안, THE 플러그인 SHALL 사이드바_패널에 "요약 중..." 로딩 상태를 표시한다
6. IF AWS_Bedrock 요약 요청이 실패하면, THEN THE 플러그인 SHALL "요약 생성에 실패했습니다. 다시 시도해주세요"라는 오류 메시지를 표시하고 영상 임베딩만 포함된 노트를 유지한다

### 요구사항 6: 요약 프롬프트 관리

**사용자 스토리:** 사용자로서, 요약 스타일을 내 취향에 맞게 커스터마이즈하고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. THE 플러그인_설정 SHALL 요약_프롬프트를 편집할 수 있는 텍스트 영역을 제공한다
2. THE 플러그인 SHALL 기본 요약_프롬프트로 `system_prompt.md` 파일의 내용을 내장하여 별도 설정 없이 바로 사용할 수 있도록 한다 (장르 자동 감지, 장르별 요약 전략, 구조화된 출력 형식 포함)
3. WHEN 사용자가 요약_프롬프트를 수정하면, THE 플러그인_설정 SHALL 변경된 프롬프트를 영구적으로 저장한다
4. THE 플러그인_설정 SHALL AWS_리전을 설정할 수 있는 옵션을 제공한다
5. THE 플러그인_설정 SHALL Bedrock_모델_ID를 설정할 수 있는 옵션을 제공한다
6. THE 플러그인 SHALL 기본 AWS_리전으로 "us-east-1"을 사용한다
7. THE 플러그인 SHALL 기본 Bedrock_모델_ID로 "anthropic.claude-3-sonnet-20240229-v1:0"을 사용한다

### 요구사항 7: 노트 저장 경로 설정

**사용자 스토리:** 사용자로서, 요약 노트가 저장되는 폴더를 지정하고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. THE 플러그인_설정 SHALL 요약_노트의 저장 폴더 경로를 설정할 수 있는 옵션을 제공한다
2. THE 플러그인 SHALL 기본 저장 경로로 "YouTube Summaries" 폴더를 사용한다
3. IF 지정된 저장 폴더가 존재하지 않으면, THEN THE 플러그인 SHALL 해당 폴더를 자동으로 생성한다

### 요구사항 8: 요약 진행 상태 피드백

**사용자 스토리:** 사용자로서, 요약 과정이 어디까지 진행되었는지 알고 싶다.

#### 인수 조건 (Acceptance Criteria)

1. WHEN 요약_버튼이 클릭되면, THE 플러그인 SHALL 요약_버튼을 비활성화하여 중복 요청을 방지한다
2. WHILE 요약 프로세스가 진행 중인 동안, THE 사이드바_패널 SHALL 현재 진행 단계를 표시한다 (예: "메타데이터 가져오는 중...", "자막 가져오는 중...", "AWS Bedrock 요약 생성 중...")
3. WHEN 요약 프로세스가 완료되면, THE 플러그인 SHALL 요약_버튼을 다시 활성화하고 유튜브_링크_입력창을 초기화한다
4. WHEN 요약 프로세스가 완료되면, THE 플러그인 SHALL "요약이 완료되었습니다"라는 성공 알림을 옵시디언 알림으로 표시한다
